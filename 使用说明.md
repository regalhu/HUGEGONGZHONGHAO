# 胡哥说餐饮公众号文章生成器使用说明

这套工具用于在本地电脑上生成「胡哥说餐饮」公众号文章、封面图和公众号排版，并可以通过微信公众平台接口上传到公众号草稿箱。

## 1. 工具能做什么

- 自动抓取昨天的餐饮相关热点，来源包括新闻、公开网页、公众号搜索、头条搜索和可抓取社交媒体线索。
- 按引用频次统计排名前 10 的餐饮相关热点关键词，并分别生成公众号文章。
- 标题末尾自动使用连续自然数编号。
- 自动生成公众号封面图。
- 每篇公众号文章固定 1 张封面图，正文保留 `[图片插入位]` 方便后续手动替换。
- 自动生成公众号 HTML 排版。
- 在浏览器后台预览文章、封面图和排版。
- 点击按钮上传单篇或一批文章到公众号草稿箱。
- 在浏览器里调整文章类型、期号，并查看最近 5 天餐饮高频关键词。
- 对生成的本地草稿进行单篇归档、单篇删除、全部归档、全部删除，也可以勾选多篇后批量处理。
- 在文章预览页生成封面图提示词，复制到 ChatGPT Plus 后手动生成封面图。

## 2. 重要说明

这套工具是本地运行工具，GitHub 只用来保存代码。

公众号的 AppID、AppSecret、OpenAI API Key、生成记录、草稿记录都保存在本机，不会上传到 GitHub。

目前图片生成支持两种方式：

- 本地漫画图：不需要 OpenAI Key，稳定可用。
- OpenAI API 图片：需要 `OPENAI_API_KEY`。

不能直接通过“登录 ChatGPT 网页账号”来稳定自动生成图片。ChatGPT 网页账号和 OpenAI API 是两套体系；如果没有 API Key，建议先使用本地漫画图。

## 3. 启动网页后台

打开 PowerShell，执行：

```powershell
cd "C:\Users\regal\Documents\胡哥说餐饮"
$env:PYTHONPATH="src"
.\.venv\Scripts\python.exe -m huge_catering.webapp --host 127.0.0.1 --port 8765
```

如果 8765 被占用，可以换成 8766 或 8767：

```powershell
$env:PYTHONPATH="src"
.\.venv\Scripts\python.exe -m huge_catering.webapp --host 127.0.0.1 --port 8767
```

然后在浏览器打开：

```text
http://127.0.0.1:8765
```

如果你用的是 8767，就打开：

```text
http://127.0.0.1:8767
```

本机已配置开机/登录后自动启动网页后台，启动脚本是：

```text
scripts/start_web.ps1
```

Windows 启动文件夹里放置的是英文名隐藏启动入口：

```text
huge-catering-web.vbs
huge-catering-web.cmd
```

如果网页打不开，可以手动运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File "C:\Users\regal\Documents\胡哥说餐饮\scripts\start_web.ps1"
```

## 4. 第一次配置公众号信息

项目根目录有 `.env` 文件，里面需要配置公众号后台信息：

```dotenv
WECHAT_APP_ID=你的公众号AppID
WECHAT_APP_SECRET=你的公众号AppSecret
BRAND_NAME=胡哥说餐饮
COLUMN_NAME=餐饮要赚钱 听我10句劝
AUTHOR_NAME=胡哥
OUTPUT_DIR=outputs
TOPIC_LIBRARY=data/topic_library.json
START_ISSUE_NUMBER=229
ENABLE_TREND_CONTENT=true
```

注意：

- AppSecret 如果看不到，只能在公众号后台重置。
- 微信公众平台后台需要把当前电脑的公网 IP 加入开发者配置的 IP 白名单。
- `.env` 不会上传到 GitHub。

## 5. 主页面生成设置

网页后台顶部有两个入口：

- 主页面
- 归档列表

生成设置已经合并到主页面顶部，可以直接调整这些内容：

### 文章设定

- 文章类型：
  - 餐饮10句劝：标题固定为 `餐饮要赚钱 听我10句劝｜第X期`
  - 餐饮热点解读：标题自动带 `#热点话题#` 和时间标记
  - 餐饮方法论：标题自动带 `#热点话题#` 和时间标记
- 期号/标题：
  - 10句劝可填写起始期号，例如 `229`
  - 留空时系统自动递增
- 关键词抓取结果：
  - 系统自动展示最近 5 天公开平台和搜索引擎里的餐饮高频词。
  - 页面会显示关键词排名、提及率和数据源范围。
- 核心生成逻辑：
  - 1000 字以内。
  - 强实用性。
  - 措辞幽默。
  - 引用公开数据源。
  - 每条都落到门店可执行动作。

设置保存后，从下一次生成文章开始生效。

## 6. 生成文章预览

在主页面：

1. 选择文章类型。
2. 填写生成篇数，默认 5 篇，最多 10 篇。
3. 点击「一键生成」或「一键重新生成」。

生成完成后，页面会显示每篇文章卡片，包括：

- 标题
- 日期
- 选题
- 关键词
- 本篇对应的热点词和引用频次
- 是否已经上传公众号草稿箱

点击「预览」可以查看：

- 封面图
- 正文排版

## 7. 上传到公众号草稿箱

预览确认没问题后，有两种上传方式：

- 单篇上传：在某篇文章卡片上点击「上传」。
- 批量上传：点击「上传本批次到公众号草稿箱」。

上传成功后，页面会显示草稿 `media_id`。

上传的是公众号后台草稿，不是直接群发。建议你最后进入公众号后台再人工检查一遍。

## 8. 草稿归档和删除

主页面支持这些本地管理功能：

- 单篇归档：把某一篇从当前主页面移到归档列表。
- 单篇删除：删除某一篇本地生成文件。
- 勾选多篇后归档：批量移动到归档列表。
- 勾选多篇后删除：批量删除本地生成文件。
- 全部归档：把当前列表里的草稿全部归档。
- 全部删除：把当前列表里的草稿全部删除。

归档文件会移动到：

```text
outputs/_archive/YYYY-MM-DD/
```

注意：这些按钮只管理本地生成文件。如果文章已经上传到微信公众号草稿箱，本地删除不会删除公众号后台里的远程草稿，需要你进入公众号后台手动删除。

## 9. 封面工具

每篇文章的预览页都有「图片生成工作台」。

系统会自动生成：

- 公众号封面图，推荐比例 2.35:1

提示词包含：

- 图片用途
- 推荐尺寸比例
- 中文提示词
- 英文提示词
- 风格说明
- 禁止元素

使用方式：

1. 在主页面找到对应草稿，点击「预览」。
2. 点击某一组旁边的「复制提示词」。
3. 点击「打开 ChatGPT 图片生成」。
4. 在 ChatGPT Plus 里粘贴提示词，手动生成图片。
5. 回到预览页替换封面图。

当前版本为低成本人工协作模式，不接入 OpenAI API，不需要 API Key，不产生 token 费用。代码中已预留 `imageProvider=manual_chatgpt`，后续可以升级为 API 自动生成版本。

每篇公众号文章固定 1 张封面图。再次上传公众号草稿箱时，系统只上传这一张封面图，正文保持纯文字排版。

## 11. 生成文件在哪里

每篇文章的生成结果保存在：

```text
outputs/YYYY-MM-DD/
```

里面包含：

```text
article.html      公众号正文排版
cover.jpg         封面图
metadata.json     文章标题、关键词、编号、草稿ID等信息
```

`metadata.json` 里还有 `image_check` 字段。现在公众号正文只保留 `[图片插入位]` 文本占位，不直接引用图片，正常情况下：

```text
ok=true
local_images=[]
remote_images=[]
```

## 12. 编号规则

标题末尾使用自然数编号。

起始编号在 `.env` 里配置：

```dotenv
START_ISSUE_NUMBER=229
```

生成记录保存在：

```text
data/publish_history.json
```

同一天重复生成时，会尽量沿用当天编号，避免编号乱跳。

## 13. 避免内容重复

系统会记录已经生成过的选题和标题：

```text
data/publish_history.json
```

热点生成启用时，系统会结合最近公开平台和搜索引擎中的餐饮高频词重新生成文章。

本地内容库模式下，10篇文章用完后，会自动生成新一组内容，并避开历史记录里已经用过的选题。

## 15. 常见问题

### 网页打不开

确认 PowerShell 里的服务还在运行。

如果 8765 被占用，换端口：

```powershell
$env:PYTHONPATH="src"
.\.venv\Scripts\python.exe -m huge_catering.webapp --host 127.0.0.1 --port 8767
```

然后打开：

```text
http://127.0.0.1:8767
```

### 不能上传公众号草稿

检查：

- `.env` 里是否有 `WECHAT_APP_ID`
- `.env` 里是否有 `WECHAT_APP_SECRET`
- 公众号后台 IP 白名单是否加了当前电脑公网 IP
- VPN 是否导致出口 IP 变化

### OpenAI 图片不能生成

检查：

- 是否填写了 `OPENAI_API_KEY`
- 是否在主页面生成设置里选择了 OpenAI 图片生成
- API Key 是否有可用额度

如果没有 API Key，使用本地漫画图即可。

### 生成内容重复

可以先删除或备份不想保留的测试输出，但不要随意删除 `data/publish_history.json`，否则系统会忘记历史编号和历史选题。

### 想重新生成某一天

可以在网页里选择同一天重新生成，也可以删除对应目录后再生成：

```text
outputs/YYYY-MM-DD/
```

如果当天已经写入历史记录，编号通常会沿用原来的编号。

## 16. 推荐日常流程

1. 打开 PowerShell 启动网页后台。
2. 浏览器进入主页面。
3. 在主页面确认文章类型、期号和关键词抓取结果。
4. 在主页面生成 5 篇预览，必要时一键重新生成。
5. 逐篇预览，确认标题、正文和封面图。
6. 点击上传单篇或批量上传。
7. 进入公众号后台草稿箱做最终检查。
8. 手动发布或定时发布。
